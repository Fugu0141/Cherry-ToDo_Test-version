# Code of Conduct

Cherry is intended to be a friendly open-source project.

This code of conduct is intentionally simple while the project is small.

---

## Expected behavior

Please:

- be respectful
- assume good intent when possible
- give constructive feedback
- explain technical disagreements calmly
- keep issues and pull requests focused
- welcome beginners and first-time contributors

---

## Unacceptable behavior

Please do not:

- harass, insult, or threaten others
- attack people instead of ideas
- post discriminatory or hateful comments
- spam issues or pull requests
- intentionally derail discussions
- submit copied code, copied assets, or copied UI from another product

---

## Reporting problems

If a discussion becomes uncomfortable or harmful, please contact the repository owner or open a private communication channel if available.

For now, the maintainer may remove comments, close issues, or block users when necessary to keep the project safe.

---

## Project culture

Cherry should be a place where small experiments, careful design, and respectful open-source development can happen without pressure.
