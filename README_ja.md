# Cherry

[English](./README.md) | [日本語](./README_ja.md)

Cherry は、目標・実行する作業・分岐・予定をつながった流れとして整理する、オープンソースのタスクフロー型計画アプリです。

平面的なリストやカレンダーから始めるのではなく、まず目標を置き、それを実行可能な作業へ分解し、見える流れとしてつなげていきます。ボード表示は流れを組み立てて理解するための画面、リスト表示は今や近いうちに注意すべき作業を見つけるための画面です。

> 現在の状態: v0.1 公開プロトタイプ基準 / v0.2 基盤開発中
>
> リポジトリ名: `Cherry-ToDo`

## 製品の方向性

```text
まず、やることの流れを作る。
日付などの情報は、役立つときに加える。
```

Cherry は次のルールを大切にします。

- タスク同士の関係を中心構造とし、予定は独立した補助レイヤーとして扱います。
- 目標タスクは文脈を与え、実行タスクは実際に行う作業を表します。
- ボード上の座標を、優先度・日付・流れの順序へ勝手に変換しません。
- 日付レーンを隠しても日付を削除せず、表示しても日付未設定タスクへ勝手に日付を付けません。
- 日付未設定は「今日」ではなく、日付未設定のまま扱います。
- モバイルは単なる小さいデスクトップではなく、別の操作設計として作ります。
- ローカルデータはユーザーが管理でき、外部へ書き出せる状態を保ちます。

製品と目標アーキテクチャの正式な要件は [`docs/REQUIREMENTS.md`](docs/REQUIREMENTS.md) にあります。

## 現在のプロトタイプで使える機能

- 目標 / タスクの作成と、`+` ハンドルからの子タスク作成
- 同じ流れへの追加と分岐の作成
- 親子フローのリンク表示と関係を考慮したレイアウト
- デスクトップでのハンドルドラッグによる既存タスク同士の接続
- 日付レーンと、ドラッグ & ドロップによる明示的な日付変更
- 同じタスクデータを使うボード表示と実行リスト表示
- todo / done 切り替え、編集、削除、Undo
- 自動整列と縦整列
- Start ページと複数のワークスペースタブ
- タブの表示、名前変更、複製、削除、タブ列内での新規作成
- Start・ワークスペース・初回保存方法選択を、最終画面の表示前に決める明示的な起動ルーティング
- 選択した画面の準備が完了するまで未完成のアプリ画面を見せない起動・読み込み・復旧状態
- 初回に選べる、ブラウザへの永続保存とメモリ内だけの一時セッション
- 保存済み参照が有効な場合に、最後の Start / ワークスペース状態・開いていたタブ・ボード / リスト表示を安全に復元
- 日本語 / 英語 UI
- 初回紹介画面と操作チュートリアル
- 自動・ライト・ダークテーマ
- 旧ストレージキーや既存プロトタイプデータとの互換性
- 暗号化された `.cherry` ワークスペースのインポート / エクスポート
- 基本的な iCalendar VTODO（`.ics`）のインポート / エクスポート
- モバイル向け縦型ボード、選択タスク用アクションドック、ボトムシート、Flow Map / ミニマップ
- タスク移動・接続中のデスクトップ端スクロール

## ローカルでの実行

Cherry はビルド不要の静的 Web アプリです。ブラウザ機能やファイル操作を安定して使うため、`index.html` を直接開くよりローカル静的サーバーの利用を推奨します。

```bash
python -m http.server 8000
```

その後、次の URL を開きます。

```text
http://localhost:8000/
```

基本機能にアカウントやネットワーク接続は必要ありません。初回利用時に、作業をブラウザへ永続保存するか、一時セッションとして使うかを選べます。一時セッションのデータはメモリ内に保持され、ページを閉じたり再読み込みしたりした後に残ることを意図していません。持ち運べるバックアップには暗号化 `.cherry` ファイルを使えますが、パスフレーズを忘れた場合は復元できません。

## 開発メモ

Cherry はもともと `Fugu0141.github.io/ToDo` で開発され、オープンソース開発のために独立したリポジトリへ移されました。

一部のファイル名、URL、リポジトリ名、保存キー、互換コードには `Cherry-ToDo` や古い名前が残っています。ユーザー向けのアプリ名と現在のドキュメントでは `Cherry` を使います。

現在のプロトタイプはスクリプト順序と互換レイヤーに依存しています。新しい広範囲な上書きスクリプトは追加せず、実装計画に従って一つずつ責務を分離してください。

挙動を変更する前に [`docs/MANUAL_TEST_CHECKLIST.md`](docs/MANUAL_TEST_CHECKLIST.md) を確認してください。既知の制限は [`docs/KNOWN_ISSUES.md`](docs/KNOWN_ISSUES.md) で管理しています。
