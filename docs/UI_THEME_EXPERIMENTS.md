# UI Theme Experiments

Cherry のライト/ダークテーマとモバイルUIを、場当たり的な色変更ではなく、仮説検証で改善するためのメモです。

## 背景

Issue #64 では、モバイルUIの色数が多く、タスク・接続線・日付レーン・操作ボタン・ミニマップが同時に強く主張していることが問題になりました。

その後、色を抑える方向で修正したところ、今度は次の問題が出ました。

- ライトテーマが白っぽく、ブロックの境界が曖昧に見える
- ダークテーマが暗いだけに見え、Cherryらしい軽さやタスクの視認性が弱い
- 接続線、選択枠、カード、レーンの優先順位が安定しない
- テーマ切替ボタンがツールバー上で主操作と混ざり、UI密度を上げている
- 既存CSSに対して上書きが増え、どの変更が何を改善したのか分かりづらい

このため、以後は「見た目を少しずつ触る」のではなく、観察・仮説・検証の単位で進めます。

## Cherryで守りたい優先順位

Cherryの画面でユーザーが見たい情報の優先順位は、以下の順番だと考えます。

1. 選択中のタスク
2. タスクカード
3. タスク同士の流れ
4. 日付レーン
5. 操作ボタン
6. ミニマップなど補助UI

つまり、線やボタンがカードより目立つ状態は避けます。一方で、カードとレーンの境界が弱すぎると構造が分からなくなるため、色数を減らすだけでは不十分です。

## デザイン原則

### 1. 色数よりも役割を制限する

色を増やす/減らすこと自体ではなく、色の役割を固定します。

- アクセント色: 選択中、追加、現在注目している流れ
- 危険色: 削除、リセット、期限超過など
- 日付色: 今日・日付レーン
- 紙色: タスクカード
- ニュートラル色: 背景、通常線、未選択UI

### 2. 明度差で面を分ける

ライトテーマでは、背景・ボード・レーン・カードが同じ白系に寄ると単調になります。

ダークテーマでは、背景・ボード・カードが暗い色に寄ると重くなります。

どちらも、色相よりも明度差でレイヤーを分けます。

### 3. タスクカードを主役にする

Cherryは「流れをブロックで育てる」UIなので、カードが置かれている感覚が重要です。

カードには次の要素を持たせます。

- 背景から切り出される色
- 控えめだが見える枠線
- ほんの少しの影
- 状態を示す左アクセント

### 4. 接続線は説明役に留める

接続線は構造を伝えるために必要ですが、常にカードより強く見えるべきではありません。

- 通常線は細く低彩度
- 選択中の経路だけ少し強める
- 破線は通常線より弱くする
- 選択中カードの枠より線を強くしない

### 5. テーマ切替はタスク操作と分ける

テーマ切替は頻繁に使うタスク操作ではなく、環境設定に近い操作です。

ルート追加・整列・削除と同じ強さで並べると、主操作を邪魔します。

今後は、ツールバー直置きではなく、小さな設定操作として扱う方向を検討します。

## 仮説

### 仮説A: ライトテーマは「白UI」より「紙カード + 低彩度キャンバス」が合う

ライトテーマを白くしすぎると、情報は読めてもUIとして単調に見えます。

検証する状態:

- 背景: わずかに青みのあるニュートラル
- ボード: 背景より少し明るい面
- 日付レーン: 低彩度の薄い青緑
- タスクカード: 淡い紙色
- 接続線: グレー寄り
- 選択中だけ紫

観察ポイント:

- 最初にタスクカードが目に入るか
- レーンとカードの境界が自然に分かるか
- 色が多すぎる印象にならないか

### 仮説B: ダークテーマでは黄色を暗くするより、温かい低彩度カードを置く方が見やすい

黄色カードをそのまま暗くすると、茶色っぽく濁って見えます。

検証する状態:

- 背景: 深い紺
- ボード: 背景より少し明るい紺
- カード: 暗い黄色ではなく、温かいグレージュ/クリーム系
- 文字: 純白ではなく淡いクリーム
- 接続線: 青灰色
- 選択中だけ紫

観察ポイント:

- ダークでもカードが沈まないか
- 重すぎる印象にならないか
- 長時間見ても疲れにくいか

### 仮説C: 接続線は常時強調ではなく、選択中の文脈だけ強くする方がよい

全ての接続線が強いと、タスクより線が主役になります。

検証する状態:

- 通常線: 低彩度で細め
- 選択中経路: アクセント色だがカード枠より弱い
- 同日リンクの破線: 通常線より弱い

観察ポイント:

- 流れは追えるか
- 線がうるさくないか
- 選択中カードが線より主役になっているか

### 仮説D: テーマ切替はツールバー内の長いボタンではなく、控えめな設定UIがよい

テーマ切替は重要ですが、頻繁な操作ではありません。

検証する状態:

- 表示文言を短くする
- タスク操作群と視覚的に分ける
- モバイルでは幅を取りすぎない

観察ポイント:

- ツールバーが詰まって見えないか
- タスク操作の邪魔をしないか
- 今のテーマ状態は分かるか

### 仮説E: モバイル操作UIはタスクの近くに浮かせるより、下部固定の方が流れを隠しにくい

選択タスクの直下に操作バーを出すと、子タスク・次の日付・接続線を隠してしまいます。

検証する状態:

- 操作バーは画面下部に固定
- 選択タスクが操作バーに隠れそうなときはスクロール補正する
- ボード側には下部操作バー分の余白を確保する

観察ポイント:

- 操作しようとした瞬間に流れが隠れないか
- 選択タスクと操作バーの対応関係が分かるか
- 片手操作しやすいか

### 仮説F: ミニマップは常時表示ではなく、ナビゲーション中だけ見える方が補助UIとして自然

ミニマップは全体把握の補助として有用ですが、常時表示すると上部操作やタスクを邪魔します。

検証する状態:

- 初期表示・選択中だけではミニマップを出さない
- スクロール中やミニマップ操作中だけ一時表示する
- 表示位置は下部操作バーと衝突しない場所に寄せる

観察ポイント:

- 普段の画面密度が下がるか
- 必要なときにだけ全体把握できるか
- タスクや操作ボタンを隠さないか

## 検証方法

1. 1回の変更で検証する仮説を1つに絞る
2. ライト/ダーク/モバイル/PCで同じタスク配置を確認する
3. スクリーンショットを残す
4. 下の評価表で主観評価する
5. 良ければ採用、悪ければ戻すか別案を試す

## 今回の実験

### Experiment A: Paper Canvas

検証対象: 仮説A、仮説B、仮説Cの一部

目的:

- ライトテーマを「白UI」ではなく「紙カード + 低彩度キャンバス」に寄せる
- ダークテーマでカードが濁りすぎないよう、暗い黄色ではなく温かいカード色にする
- 接続線をカードより弱くし、選択中カードを主役に戻す

実装方針:

- `theme-experiment-a.css` を `visual-cleanup.css` の後ろで読み込む
- 既存実装を完全に置き換えず、実験レイヤーとして上書きする
- 良ければ `visual-cleanup.css` に統合し、悪ければ実験CSSを外す

確認したいこと:

- ライトで単調さが減るか
- ダークでカードが沈まないか
- 接続線が主張しすぎないか
- タスクカードと選択状態の優先順位が戻るか

### Experiment B: Mobile Control Cleanup

検証対象: 仮説E、仮説F、日付ラベルの主従整理

目的:

- 操作バーが選択タスク周辺の流れを隠す問題を減らす
- ミニマップを常時表示から外し、補助UIに戻す
- 日付ラベルのカード感を弱め、タスクカードとの主従関係を戻す

実装方針:

- `mobile-control-cleanup.css` を実験CSSとして追加
- `mobile-action-bar.js` で操作バーを選択カード追従から下部ドック表示へ変更
- 選択タスクが下部ドックに隠れそうな場合はスクロール補正する
- `mobile-flow-map.js` で選択中タスクがあるだけではミニマップを常時表示しない
- スクロールやミニマップ操作中だけミニマップを一時表示する

確認したいこと:

- スマホで選択中タスクの次の流れが隠れにくくなるか
- ミニマップが邪魔な常駐UIから補助UIに戻るか
- 日付ラベルがタスクカードと競合しなくなるか
- 下部ドックが逆に画面を圧迫しすぎないか
