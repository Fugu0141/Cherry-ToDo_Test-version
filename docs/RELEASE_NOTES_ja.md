# Cherry v0.1.0 リリースノート

Cherry v0.1.0 は、最初の公開プロトタイプ版です。

Cherry は、タスクを単なるリストではなく、つながったタスクブロックの流れとして整理するためのオープンソース ToDo アプリです。このリリースでは、「まず流れを組み立ててから、今日やることを見つける」という Cherry の核になる考え方を、実際に触れる形にすることを重視しています。

## 主な内容

- タスクブロックを親子関係のある流れとしてつなげられます。
- ルートタスクを、プロジェクトや大きな見出しのように扱えます。
- 子タスクは、同じ流れとして続けることも、分岐として分けることもできます。
- 日付レーンにより、日付だけに依存せずにタスクを予定へ配置できます。
- 実行リストでは、未定・今日まで・今後のタスクをルートごとに確認できます。
- モバイル向けには、縦方向のボード表示と Flow Map / ミニマップを用意しています。
- 初回表示のウェルカム画面とチュートリアルで、Cherry の考え方と基本操作を説明します。
- 日本語と英語の2言語表示に対応しました。
- スタートページから、複数の作業タブを管理できるようにしました。
- ワークスペースを暗号化された `.cherry` ファイルとして保存・読み込みできます。

## 暗号化 Cherry ファイル形式

Cherry v0.1.0 では、すべてのタブを1つのファイルにまとめて保存できるワークスペース形式を追加しました。

`.cherry` ファイルは、暗号化された JSON 形式です。

- 形式: `cherry-workspace-encrypted`
- 暗号化: AES-GCM
- 鍵導出: PBKDF2 / SHA-256
- salt と IV: 書き出しごとにランダム生成
- 保存範囲: すべてのタブを含むワークスペース全体

読み込み時には、書き出し時に設定したパスフレーズが必要です。パスフレーズを忘れた場合、ファイルの復元はできません。

## 既知の制限

Cherry はまだ初期プロトタイプです。このリリースでは、次の点は未完成のまま残しています。

- スケジュールモデルは、古い日付文字列ベースのデータから移行途中です。
- いくつかの実装ファイルは、互換性維持や修正レイヤーとして残っています。
- 同日内のレイアウトはまだ基本的なものです。
- モバイル操作は使える状態ですが、完全な磨き込みはこれからです。
- デスクトップの作成・編集モーダルは、まだインラインエディタには置き換えていません。
- 明示的に書き出さない限り、データはブラウザ内にローカル保存されます。

## リリース前に確認すること

v0.1.0 のタグを作る前に、`docs/MANUAL_TEST_CHECKLIST.md` を確認してください。特に次の項目を重点的に確認します。

- 基本的なタスク作成・編集
- ブランチ作成
- ドラッグ & ドロップによる日付変更
- 実行リストの挙動
- モバイル表示
- localStorage 互換性
- 暗号化ファイルの保存・読み込み

## アップグレードメモ

既存のローカルタスクデータは、従来の localStorage キーから引き続き読み込める想定です。Cherry v0.1.0 では新しいタブ管理のためにワークスペース形式も作成しますが、互換性のために従来の保存キーも残します。

## このリリースの位置づけ

このリリースは完成版ではありません。Cherry の方向性を見せ、タスクフローの考え方、モバイルでの使いやすさ、実行リスト、ワークスペース管理、リリースドキュメントについてフィードバックを集めるための公開プロトタイプです。
